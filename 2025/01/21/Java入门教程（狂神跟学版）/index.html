5.3 继承

  • Java中类只有单继承没有多继承
  • 在java中，所有类，都直接或者间接继承Object
  • 私有的东西无法被继承

（1）super
super 是一个关键字，主要用于子类访问父类的成员（包括属性和方法）或调用父类的构造方法。它在继承关系中非常重要，能够帮助子类明确地与父类交互。

super 的主要用途

  1. 访问父类的成员变量：

    • 当子类与父类有同名的成员变量时，super 用于区分调用的是父类的变量，而不是子类的变量。

  2. 调用父类的方法：

    • 当子类重写了父类的方法时，可以使用 super 调用父类版本的方法。

  3. 调用父类的构造方法：

    • 使用 super(…) 显式调用父类的构造方法。必须在子类构造方法的第一行调用。

super 的注意事项

  1. 必须在构造方法的第一行使用 super：
    • 如果子类的构造方法显式调用父类构造方法，super 必须是第一行。
    • 如果子类构造方法没有显式调用 super，Java 编译器会默认调用父类的无参构造方法。
    • 如果父类没有无参构造方法，而子类没有调用父类其他构造方法，则会编译错误。
  2. 不能在静态方法中使用 super：
    • super 是用来访问对象的，而静态方法属于类本身，与对象无关，因此不能在静态方法中使用 super。

（2）方法重写
定义:方法重写（Method Overriding） 是指子类提供了一个与父类中方法具有相同的方法名、参数列表和返回类型（从 Java 5 开始允许协变返回类型）的实现，以覆盖父类中的方法。方法重写的主要目的是让子类能够根据自身的需要对父类方法进行重新定义，从而实现多态。

方法重写的关键点

  1. 方法签名必须相同：
    • 方法名、参数列表必须与父类中被重写的方法一致。
    • 返回值必须相同或是父类方法返回值的子类（即协变返回类型）。
  2. 访问修饰符：
    • 子类重写的方法的访问修饰符不能比父类的更严格。例如，父类方法是 protected，子类不能用 private。
    • 子类可以使用更宽松的修饰符，例如将 protected 改为 public。
  3. 不能重写 final 方法：
    • 如果父类的方法被声明为 final，则子类不能重写该方法。
    • 不能重写 static 方法：static 方法是与类绑定的，而不是与实例绑定，因此不能被重写，但可以在子类中定义一个相同签名的 static 方法（这种情况称为隐藏，而不是重写）。
  4. @Override 注解：
    • 推荐在重写的方法前加上 @Override 注解，这样编译器会检查是否正确地完成了重写。
      调用父类方法：
  5. 可以通过 super.方法名 调用父类的被重写方法。

方法重写与方法重载的区别

5.4 多态

多态（Polymorphism） 是面向对象编程的重要特性之一，指同一个方法在不同对象中可以表现出不同的行为形式。多态的核心是通过父类引用指向子类对象，从而实现行为的动态决定。它提高了代码的可扩展性和灵活性。

多态的两种类型

  1. 编译时多态（静态多态）：

    • 通过方法重载（Method Overloading）实现。
    • 方法的调用在编译时确定，取决于方法参数的数量或类型。
      运行时多态（动态多态）：

  2. 通过方法重写（Method Overriding）实现。

    • 方法的调用在运行时决定，取决于引用变量所指向的实际对象。

多态的实现条件

  1. 继承：必须有父类和子类。
  2. 方法重写：子类需要重写父类的方法。
  3. 父类引用指向子类对象：通过这种方式调用子类重写的方法。

5.6 static

static 是一个关键字，表示“静态”的含义。它可以用于修饰变量、方法、代码块和内部类，主要作用是让成员属于类本身，而不是特定的实例对象。

static 的特点

  1. 静态成员属于类本身：
    • 不需要创建类的对象即可通过类名直接访问静态变量和方法。
  2. 内存中只存在一份：
    • 静态变量或方法在内存中只加载一次，被所有对象共享，节省内存空间。
  3. 与实例无关：
    • 静态成员可以在没有创建对象的情况下访问，但它无法直接访问非静态成员（需要通过实例对象）。

static 的主要用途

  1. 静态变量
    静态变量也称为类变量，属于整个类，而不是某个对象。
  1. 静态方法
    静态方法可以通过类名直接调用，通常用于不需要依赖实例变量的操作。
  2. 静态代码块
    静态代码块用于对类的静态变量进行初始化，只会执行一次，当类第一次加载时运行。

5.7 抽象类

抽象类（abstract class）是一种不能直接实例化的类，通常用于定义某种通用的概念或行为，为子类提供基础。抽象类可以包含抽象方法和非抽象方法，目的是让子类继承并实现特定的行为。

抽象类的特点

  1. 无法实例化：

    • 抽象类不能被直接创建对象，只能通过其子类创建实例。

  2. 包含抽象方法：

    • 抽象类可以包含抽象方法（没有方法体的方法），需要子类实现。
    • 抽象方法的声明：abstract 返回值类型 方法名(参数列表);

  3. 可以包含非抽象方法：

    • 抽象类可以包含带有方法体的普通方法，这些方法可以直接被子类继承或重写。

  4. 可以包含成员变量：

    • 抽象类可以定义成员变量，子类可以直接使用或覆盖。

  5. 可以有构造方法：

    • 虽然抽象类不能实例化，但它可以定义构造方法，供子类调用以初始化抽象类中的成员。

5.8 接口

接口（interface）是一个特殊的结构，用于定义一组方法规范或规则，规定类必须提供这些方法的具体实现。接口是面向对象编程中实现多态和多继承的重要工具。

接口的特点

  1. 只定义规范，不提供实现：

    • 接口中的方法默认是 abstract（抽象的），不包含方法体（从 Java 8 开始，可以定义默认方法和静态方法）。

  2. 支持多继承：

    • 一个类可以实现多个接口，从而避免了 Java 单继承的限制。

  3. 成员默认修饰符：

    • 接口中的方法默认是 public abstract。
    • 接口中的变量默认是 public static final（常量）。

  4. 不能包含实例变量和构造方法：

    • 接口只能定义常量，不能定义实例变量，也不能有构造方法。

  5. 接口本身可以继承：

    • 接口可以继承另一个接口，并且支持多继承。

5.9 内部类

内部类（Inner Class）是指在另一个类的内部定义的类。

  1. 成员内部类（非静态内部类）
    • 成员内部类是外部类的一个成员，可以定义成员变量和方法。
    • 它与外部类实例关联，只有通过外部类的实例才能创建成员内部类的对象。
  1. 静态内部类
    • 使用 static 修饰，属于外部类本身，而不是外部类的实例。
    • 可以直接通过外部类访问静态内部类，而无需外部类实例。
    • 静态内部类只能访问外部类的静态成员。
  1. 局部内部类（定义在方法或代码块中）
    • 定义在方法或代码块中，作用范围仅限于方法或代码块内。
    • 局部内部类可以访问外部类的成员和方法作用域内的局部变量（必须是 final 或隐式 final 的）。
  1. 匿名内部类（没有名字的内部类）
    • 匿名内部类没有名字，通常用来简化代码。
    • 一般用于继承某个类或实现某个接口，并且只需使用一次。
    • 使用场景：事件处理、回调等。

6、 异常机制

异常的分类
Java 中的异常继承自 java.lang.Throwable，分为两大类：

  • Error（错误）：

    • 继承自 Throwable。
    • 表示 JVM 无法处理的严重错误，如内存溢出（OutOfMemoryError）。
    • 程序一般不需要也无法处理。
    • 示例：StackOverflowError、VirtualMachineError。

  • Exception（异常）：

    • 继承自 Throwable。
    • 表示程序中可以处理的错误情况。
    • 又分为：
      • 受检异常（Checked Exception）：
        在编译时必须被处理（通过 try-catch 或 throws）。
        示例：IOException、SQLException。
      • 非受检异常（Unchecked Exception）：
        在运行时抛出，不强制要求处理。
        示例：NullPointerException、ArrayIndexOutOfBoundsException。

异常处理

  1. 检测异常：当程序运行中发生异常情况时，Java 虚拟机会抛出异常。
  2. 抛出异常：使用 throw 抛出异常对象。
  3. 捕获异常：使用 try-catch 捕获并处理异常。
  4. 清理代码：通过 finally 块执行一些必要的清理工作。

异常的关键字

  1. try-catch-finally
    try 块：包含可能会抛出异常的代码。
    catch 块：捕获并处理特定类型的异常。
    finally 块：无论是否发生异常都会执行的代码（常用于释放资源）。
