知识补充
在 Java 中，HashSet 是一个基于哈希表实现的集合类，用于存储不重复的元素。它是 Set 接口的常用实现类之一，具有高效的查找、插入和删除操作。

HashSet 的特点

  1. 不允许重复元素：HashSet 中的元素是唯一的，重复元素会被自动忽略。
  2. 无序性：HashSet 不保证元素的存储顺序，元素的顺序可能与插入顺序不同。
  3. 允许 null 值：HashSet 可以存储一个 null 值。
  4. 基于哈希表实现：底层使用 HashMap 来存储元素，因此查找、插入和删除操作的时间复杂度接近 O(1)。
  5. 非线程安全：HashSet 不是线程安全的，多线程环境下需要额外同步。

HashSet 的常用方法
以下是 HashSet 的常用方法：

方法 描述
add(E e) 添加元素，如果元素已存在则返回 false。
remove(Object o) 删除指定元素，如果元素存在则返回 true。
contains(Object o) 检查集合是否包含指定元素。
size() 返回集合中元素的数量。
isEmpty() 检查集合是否为空。
clear() 清空集合中的所有元素。
iterator() 返回一个迭代器，用于遍历集合。

HashSet 的使用示例

HashSet 的底层实现
HashSet 的底层是基于 HashMap 实现的。HashSet 的元素实际上是存储在 HashMap 的键中，而值则是一个固定的常量对象（PRESENT）。

HashSet 的性能

  • 添加、删除、查找操作：时间复杂度接近 O(1)，因为底层是哈希表。
  • 遍历操作：时间复杂度为 O(n)，因为需要遍历所有元素。

HashSet 的注意事项

  1. 元素唯一性：HashSet 通过 equals() 和 hashCode() 方法判断元素是否重复。因此，存储自定义对象时，需要正确重写这两个方法。
  2. 无序性：如果需要保持插入顺序，可以使用 LinkedHashSet。
  3. 线程安全：多线程环境下，可以使用 Collections.synchronizedSet(new HashSet<>()) 或 ConcurrentHashMap.newKeySet()。

HashSet 与 LinkedHashSet 的区别

特性 HashSet LinkedHashSet
顺序 无序 保持插入顺序
性能 略高 略低（需要维护链表）
底层实现 基于 HashMap 基于 LinkedHashMap

在 Java 里，Map 是一种非常重要的数据结构，它用于存储键值对（key - value pairs），也就是将键（key）映射到值（value）。下面从基本概念、常用实现类、基本操作、遍历方式等方面详细介绍 Java 中的 Map。

基本概念

  • Map 是一个接口，它不属于 Collection 接口体系，但和 Collection 一样都是 Java 集合框架的重要组成部分。Map 接口提供了一系列操作键值对的方法，如添加、删除、查找等。

常用实现类

  1. HashMap
  • 特点：HashMap 是最常用的 Map 实现类，它基于哈希表实现，不保证键值对的顺序。它允许使用 null 作为键和值，具有较高的查找、插入和删除操作的性能，平均时间复杂度为 O(1)。

  1. TreeMap

  1. LinkedHashMap
  • 特点：LinkedHashMap 是 HashMap 的子类，它维护了一个双向链表，记录了键值对的插入顺序（或者访问顺序，取决于构造函数的参数）。因此，它可以保证遍历键值对的顺序和插入顺序一致。性能和 HashMap 相近。

基本操作

1. 添加元素
使用 put(key, value) 方法向 Map 中添加键值对。如果键已经存在，会覆盖原来的值。

2. 获取元素
使用 get(key) 方法根据键获取对应的值。如果键不存在，返回 null。

3. 删除元素
使用 remove(key) 方法根据键删除对应的键值对。

4. 判断键是否存在
使用 containsKey(key) 方法判断 Map 中是否包含指定的键。

5. 判断值是否存在
使用 containsValue(value) 方法判断 Map 中是否包含指定的值。
