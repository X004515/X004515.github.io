1. 栈和队列基础

在 Java 中，栈（Stack）和队列（Queue）是两种重要的数据结构，它们在很多场景下都有广泛的应用。下面将从基本概念、Java 中的实现以及常见操作等方面详细介绍栈和队列的基础知识。

栈（Stack）

基本概念
栈是一种遵循后进先出（Last In First Out，LIFO）原则的数据结构。这意味着最后进入栈的元素会最先被移除。可以把栈想象成一叠盘子，最后放上去的盘子会最先被拿走。

Java 中的实现
在 Java 里，有多种方式可以实现栈，常见的有使用 Stack 类和 Deque 接口。

  • 使用 Stack 类：Stack 类是 Java 提供的一个标准类，它继承自 Vector 类。不过，由于 Stack 类是线程安全的，在性能上可能会有一些开销，而且其设计存在一些历史遗留问题，现在更推荐使用 Deque 接口来实现栈。
  • 使用 Deque 接口：Deque 是双端队列接口，它可以从两端进行元素的插入和删除操作。通过只使用一端的操作，就可以将其当作栈来使用。常用的实现类有 LinkedList 和 ArrayDeque。

常见操作

  • 入栈（Push）：将元素添加到栈的顶部。
  • 出栈（Pop）：移除并返回栈顶的元素。
  • 查看栈顶元素（Peek）：返回栈顶的元素，但不移除它。
  • 判断栈是否为空（isEmpty）：检查栈中是否有元素。

队列（Queue）

基本概念
队列是一种遵循先进先出（First In First Out，FIFO）原则的数据结构。这意味着最先进入队列的元素会最先被移除。可以把队列想象成排队买票的队伍，先到的人会先买到票。

Java 中的实现
在 Java 中，队列通过 Queue 接口来表示，常见的实现类有 LinkedList、ArrayDeque 和 PriorityQueue 等。

常见操作

  • 入队（Offer）：将元素添加到队列的尾部。
  • 出队（Poll）：移除并返回队列的头部元素。如果队列为空，则返回 null。
  • 查看队首元素（Peek）：返回队列的头部元素，但不移除它。如果队列为空，则返回 null。
  • 判断队列是否为空（isEmpty）：检查队列中是否有元素。

ArrayDeque

ArrayDeque 是 Java 集合框架中的一个类，位于 java.util 包下，它实现了 Deque 接口，是一种基于数组实现的双端队列（Double Ended Queue）。下面从基本概念、特点、常用方法、使用示例以及与其他类似数据结构的比较等方面详细介绍 ArrayDeque。

基本概念
双端队列允许在队列的两端（队首和队尾）进行元素的插入和删除操作。ArrayDeque 使用数组作为底层数据结构来存储元素，并且会根据需要自动调整数组的大小以适应元素数量的变化。

特点

  1. 高效的插入和删除操作：在队首和队尾进行插入和删除操作的时间复杂度都是 $O(1)$，因为它可以直接通过数组的索引来访问和修改元素。
  2. 无容量限制：虽然 ArrayDeque 基于数组实现，但它会在元素数量超过数组容量时自动扩容，因此理论上可以存储任意数量的元素。
  3. 不允许存储 null 元素：如果尝试向 ArrayDeque 中添加 null 元素，会抛出 NullPointerException 异常。
  4. 非线程安全：ArrayDeque 不是线程安全的，不适合在多线程环境下使用。如果需要在多线程环境中使用双端队列，可以考虑使用 ConcurrentLinkedDeque。

常用方法

插入元素

  • addFirst(E e)：在队首插入指定元素，如果队列已满（这种情况通常不会发生，因为会自动扩容），则抛出 IllegalStateException 异常。
  • addLast(E e)：在队尾插入指定元素，如果队列已满，同样抛出 IllegalStateException 异常。
  • offerFirst(E e)：在队首插入指定元素，如果插入成功则返回 true，否则返回 false。
  • offerLast(E e)：在队尾插入指定元素，如果插入成功则返回 true，否则返回 false。

删除元素

  • removeFirst()：移除并返回队首元素，如果队列为空，则抛出 NoSuchElementException 异常。
  • removeLast()：移除并返回队尾元素，如果队列为空，则抛出 NoSuchElementException 异常。
  • pollFirst()：移除并返回队首元素，如果队列为空，则返回 null。
  • pollLast()：移除并返回队尾元素，如果队列为空，则返回 null。

获取元素

  • getFirst()：返回队首元素，但不移除，如果队列为空，则抛出 NoSuchElementException 异常。
  • getLast()：返回队尾元素，但不移除，如果队列为空，则抛出 NoSuchElementException 异常。
  • peekFirst()：返回队首元素，但不移除，如果队列为空，则返回 null。
  • peekLast()：返回队尾元素，但不移除，如果队列为空，则返回 null。

其他方法

  • size()：返回队列中的元素数量。
  • isEmpty()：判断队列是否为空。
  • contains(Object o)：判断队列中是否包含指定元素。

LinkedList

LinkedList 是 Java 集合框架中的一个重要类，它位于 java.util 包下，实现了 List、Deque 等多个接口，是一种基于双向链表实现的数据结构。以下将从基本概念、特点、常用方法、使用示例以及与其他类似数据结构的比较等方面详细介绍 LinkedList。

基本概念
LinkedList 使用双向链表来存储元素，每个节点（元素）包含三个部分：指向前一个节点的引用、当前节点存储的数据以及指向后一个节点的引用。这种结构使得 LinkedList 在插入和删除元素时具有较高的效率，尤其是在链表的头部和尾部进行操作。

特点

  1. 动态大小：LinkedList 的大小可以动态变化，不需要在创建时指定固定的容量。当添加或删除元素时，链表会自动调整大小。
  2. 高效的插入和删除：在链表的任意位置插入或删除元素的时间复杂度为 $O(1)$（前提是已经定位到要操作的位置）。特别是在链表的头部和尾部进行插入和删除操作时，效率非常高。
  3. 顺序访问效率高：如果需要按顺序访问链表中的元素，LinkedList 可以通过遍历链表来实现，效率较高。
  4. 随机访问效率低：由于 LinkedList 是基于链表实现的，随机访问元素时需要从头节点或尾节点开始遍历链表，直到找到目标元素，因此随机访问的时间复杂度为 $O(n)$，效率较低。
  5. 允许存储 null 元素：LinkedList 允许存储多个 null 元素。
  6. 非线程安全：LinkedList 不是线程安全的，如果需要在多线程环境中使用，可以使用 Collections.synchronizedList() 方法将其转换为线程安全的列表，或者考虑使用 CopyOnWriteArrayList。

常用方法

添加元素

  • add(E e)：将指定元素添加到链表的尾部。
  • add(int index, E element)：在指定索引位置插入指定元素。
  • addFirst(E e)：将指定元素添加到链表的头部。
  • addLast(E e)：将指定元素添加到链表的尾部，等同于 add(E e)。

删除元素

  • remove()：移除并返回链表的头部元素。
  • remove(int index)：移除指定索引位置的元素。
  • removeFirst()：移除并返回链表的头部元素。
  • removeLast()：移除并返回链表的尾部元素。

获取元素

  • get(int index)：返回指定索引位置的元素。
  • getFirst()：返回链表的头部元素。
  • getLast()：返回链表的尾部元素。

其他方法

  • size()：返回链表中的元素数量。
  • isEmpty()：判断链表是否为空。
  • contains(Object o)：判断链表中是否包含指定元素。

使用示例

与其他类似数据结构的比较

  • 与 ArrayList 比较：ArrayList 是基于数组实现的动态数组，随机访问元素的时间复杂度为 $O(1)$，效率较高；但在插入和删除元素时，可能需要移动大量元素，时间复杂度为 $O(n)$。而 LinkedList 在插入和删除元素时效率较高，但随机访问元素的效率较低。因此，如果需要频繁进行随机访问操作，建议使用 ArrayList；如果需要频繁进行插入和删除操作，建议使用 LinkedList。
  • 与 Stack 比较：Stack 是 Java 中传统的栈实现，它继承自 Vector 类，是线程安全的。但 Stack 的性能相对较差，因为它使用了同步机制。而 LinkedList 可以作为栈来使用，并且性能更好，因为它是非线程安全的，没有同步开销。可以使用 addFirst() 和 removeFirst() 方法来模拟栈的入栈和出栈操作。
  • 与 ArrayDeque 比较：ArrayDeque 是基于数组实现的双端队列，在队首和队尾进行插入和删除操作的时间复杂度为 $O(1)$，并且随机访问元素的性能比 LinkedList 好。但 ArrayDeque 不允许存储 null 元素，而 LinkedList 允许。
