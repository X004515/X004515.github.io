Java字符串、数组、列表相互转换

发表于2025-02-11|更新于2025-02-11|Java后端
|浏览量:

在 Java 中，String、StringBuilder、StringBuffer、数组和列表（List）是常用的数据类型，它们各自有不同的特性和适用场景。以下是它们的详细比较：

1. String（字符串）

特点

  • 不可变性：String 是不可变的，一旦创建就不能修改。每次对 String 的操作（如拼接、替换）都会生成一个新的 String 对象。
  • 线程安全：由于不可变性，String 是线程安全的。
  • 性能：频繁修改 String 会导致大量临时对象的创建，性能较差。

适用场景

  • 适用于不需要频繁修改的字符串操作。
  • 适合作为常量或配置项。

示例

2. StringBuilder

特点

  • 可变性：StringBuilder 是可变的，可以在原对象上直接修改内容。
  • 非线程安全：StringBuilder 不是线程安全的，适合单线程环境。
  • 性能：由于避免了创建临时对象，StringBuilder 在频繁修改字符串时性能优于 String。

适用场景

  • 适用于单线程环境下频繁修改字符串的场景。
  • 适合拼接、插入、删除等操作。

示例

3. StringBuffer

特点

  • 可变性：StringBuffer 是可变的，可以在原对象上直接修改内容。
  • 线程安全：StringBuffer 是线程安全的，所有方法都加了 synchronized 关键字。
  • 性能：由于线程安全的开销，StringBuffer 的性能略低于 StringBuilder。

适用场景

  • 适用于多线程环境下频繁修改字符串的场景。
  • 适合需要线程安全的字符串操作。

示例

4. 数组（Array）

特点

  • 固定大小：数组的长度是固定的，创建后不能动态调整。
  • 高效访问：通过索引访问元素非常高效。
  • 类型单一：数组只能存储同一类型的元素。

适用场景

  • 适用于已知固定大小的数据集合。
  • 适合需要高效随机访问的场景。

示例

5. 列表（List，如 ArrayList）

特点

  • 动态大小：列表的长度是动态的，可以随时添加或删除元素。
  • 类型灵活：列表可以存储任意类型的对象（通过泛型指定类型）。
  • 访问效率：随机访问效率较高，但插入和删除操作可能较慢（取决于具体实现，如 ArrayList 或 LinkedList）。

适用场景

  • 适用于需要动态调整大小的数据集合。
  • 适合需要频繁插入、删除或遍历的场景。

示例

性能对比
字符串拼接性能

  • String：每次拼接都会创建新的对象，性能最差。
  • StringBuilder：性能最好，适合单线程环境。
  • StringBuffer：性能略低于 StringBuilder，但线程安全。

数据访问性能

  • 数组：随机访问性能最好，但大小固定。
  • 列表：随机访问性能较好（如 ArrayList），动态调整大小方便。

如何选择？

  • 如果字符串不需要频繁修改，使用 String。
  • 如果需要在单线程环境下频繁修改字符串，使用 StringBuilder。
  • 如果需要在多线程环境下频繁修改字符串，使用 StringBuffer。
  • 如果数据集合大小固定且需要高效访问，使用数组。
  • 如果数据集合大小不固定且需要动态调整，使用列表（如 ArrayList）。

6.4 其他常用函数

字符串操作

  • str.length()：获取字符串长度。
  • str.charAt(int index)：获取指定位置的字符。
  • str.substring(int beginIndex, int endIndex)：获取子字符串。
  • str.contains(CharSequence s)：判断字符串是否包含某个子串。
  • str.replace(CharSequence target, CharSequence replacement)：替换字符串中的字符或子串。

数组操作

  • Arrays.sort(array)：对数组进行排序。
  • Arrays.binarySearch(array, key)：在已排序的数组中进行二分查找。
  • Arrays.copyOf(array, newLength)：复制数组。

列表操作

  • list.size()：获取列表大小。
  • list.get(int index)：获取指定位置的元素。
  • list.add(E e)：添加元素。
  • list.remove(int index)：移除指定位置的元素。
  • list.contains(Object o)：判断列表是否包含某个元素。
